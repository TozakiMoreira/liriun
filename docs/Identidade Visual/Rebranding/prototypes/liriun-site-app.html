01 · Nav + Hero

Topo da página

  • Rotaapp/page.tsx · server component, hero animado em sub-component "use client".
  • NavSticky no scroll após 80 px (use useScroll do Framer Motion). Backdrop blur quando sticky.
  • Botãoshadcn Button variant="default" com nosso gradient. Hover: translateY(-1px) em 220 ms.
  • Hero h1text-[76px] font-semibold tracking-[-2.4px]. Word "conduzida por voz" usa bg-brand-grad bg-clip-text.
  • PhonesApenas decorativos. Imagens estáticas (next/image) ou fontes SVG. Animar com motion.div + parallax suave no scroll (–20 px translate).
  • Orb glowPseudo-elemento com filter: blur(20px). Pode animar opacidade 0.6→1 em loop.
  • Reduced motionuseReducedMotion() desliga parallax e badge animation.

📦 Dev handoff

Pronto pro code

Tudo neste mockup é renderizável com a stack Next.js 15 + Tailwind + Framer Motion + shadcn/ui usando os tokens do liriun-brand-kit/.

Setup
  • 1. npx create-next-app@latest liriun-web --ts --tailwind --app
  • 2. npx shadcn@latest init · escolher slate
  • 3. pnpm add framer-motion
  • 4. Cole tokens.css + shadcn-theme.css em app/globals.css
  • 5. Cole tailwind.config.snippet.js dentro de tailwind.config.ts
  • 6. Cole favicons + OG em public/
Estrutura sugerida
app/
├─ layout.tsx        ← font + metadata + OG
├─ page.tsx          ← hero, recursos, how, faq, cta
├─ precos/page.tsx
├─ globals.css       ← tokens + shadcn theme
└─ components/
   ├─ Nav.tsx
   ├─ Hero.tsx       ← "use client"
   ├─ FeatureCard.tsx
   ├─ Stats.tsx      ← motion counters
   ├─ HowStep.tsx
   ├─ PriceCard.tsx
   ├─ FaqAccordion.tsx
   └─ Footer.tsx
Princípios
  • — RSC por padrão; "use client" só onde houver motion ou state.
  • — Toda animação respeita useReducedMotion().
  • — Imagens pesadas vão por next/image com priority só no hero.
  • — Cor: nunca #hex direto, sempre bg-violet-500, text-fg-muted, etc.
  • — Sombra: só os tokens shadow-glow, shadow-lg, shadow-xl.
  • — Mobile: priority breakpoints sm 640, md 768, lg 1024, xl 1280.
