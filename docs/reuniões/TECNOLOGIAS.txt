---------- BACKEND ----------

Linguagem / Framework
  .NET 8 (C#) com ASP.NET Core Web API

Arquitetura
  Clean Architecture em camadas:
    Domain          — entidades e regras de negocio puras
    Application     — casos de uso, interfaces, DTOs
    Infrastructure  — EF Core, repositories, cliente Gemini
    API             — controllers, middleware, autenticacao

ORM / Acesso a dados
  Entity Framework Core (provider Npgsql para PostgreSQL)

Autenticacao
  JWT (JSON Web Tokens)
  BCrypt.Net ou similar para hash de senhas

Testes
  xUnit (unitarios)
  Moq ou NSubstitute (mocks)
  Testcontainers ou banco em memoria (testes de repository)


---------- FRONTEND ----------

Framework
  Angular 17+

Padroes / Recursos
  Standalone components (sem NgModules)
  Signals (para gerenciamento de estado reativo)
  Lazy loading de rotas

UI / Componentes
  PrimeNG (biblioteca de componentes)
  TailwindCSS (utility-first styling)
  Font Awesome Free (icones — REGRA: sem emojis no app)
  Inter (fonte — Google Fonts)

Cliente HTTP
  HttpClient do Angular com interceptors para JWT

Testes
  Jasmine / Karma (padrao Angular — unitarios)
  Playwright com TypeScript (E2E recomendado)
    - Roda em Chromium/Firefox/WebKit
    - Emulacao de mobile viewport nativa
    - Obrigatorio: data-testid em todos elementos interativos


---------- BANCO DE DADOS ----------

SGBD
  PostgreSQL 15+

Hospedagem
  Supabase (free tier) — PostgreSQL gerenciado na nuvem
  Alternativa futura: Postgres auto-hospedado em VM Oracle

Migrations
  EF Core Migrations


---------- INTELIGENCIA ARTIFICIAL ----------

Provedor
  Google Gemini API (free tier)

Modelo
  gemini-flash (padrao para uso leve ~3-5 tarefas/dia)

Integracao
  HTTP direto (HttpClient) OU
  Google.Cloud.AIPlatform.V1 (se disponivel pro .NET)

Formato de resposta
  JSON estruturado: { titulo, categorias[], prazo, prioridade }
  Campos null quando a IA nao consegue inferir


---------- DEPLOY E INFRAESTRUTURA ----------

Opcao A (decidir na Fase 6) — Plataformas gerenciadas
  Frontend: Vercel (free tier)
  Backend:  Railway (free tier — tem cold start de ~30s)
  Banco:    Supabase (free tier)
  Dominio:  padrao das plataformas (*.vercel.app, *.railway.app)

Opcao B (decidir na Fase 6) — Auto-hospedagem
  VM:          Oracle Cloud Free Tier always-free
               (2 VMs ARM + 200GB storage — nunca dorme)
  Container:   Docker + Docker Compose
  Reverse proxy: nginx
  SSL:         Let's Encrypt (certbot)
  Dominio:     proprio (.com.br ~R$40-60/ano)
  Frontend:    pode manter em Vercel mesmo nessa opcao

Variaveis de ambiente
  .env (local) / variaveis de plataforma (producao)
  dotenv no backend
  Angular environment.ts


---------- CI / CD ----------

Versionamento
  Git + GitHub

Pipeline (Fase 6)
  GitHub Actions
    - Build e testes a cada push
    - Deploy automatico na main (front e back)
    - Rodar suite E2E contra staging antes de promover pra producao
